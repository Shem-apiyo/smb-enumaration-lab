# Contributing to smb-enumaration-lab

We welcome contributions to the smb-enumaration-lab repository! Please follow these guidelines to make the process smooth and efficient.

## Guidelines for Contributors
- **Read the Code of Conduct**: Ensure that you understand our community guidelines.
- **Fork the Repository**: Make a personal copy of the repository by forking it.
- **Create a New Branch**: For any changes, create a new branch (e.g., `feature/my-feature`).
- **Write Clear Commit Messages**: Describe your changes in the commit messages for clarity.

## Reporting Issues
- **Check Existing Issues**: Before creating a new issue, check to see if it has already been reported.
- **Provide Detailed Information**: When reporting an issue, include:
  - Description of the issue
  - Steps to reproduce
  - Expected outcome vs. actual outcome
  - Screenshots if applicable

## Pull Request Process
1. **Create a Pull Request**: Once your changes are ready, push them to your fork and create a pull request against the `main` branch of the original repository.
2. **Describe Your Changes**: Provide a clear description of what changes were made and why.
3. **Respond to Reviewer Feedback**: Be open to feedback from maintainers and be prepared to make adjustments.
4. **Merge the Pull Request**: Once approved, you can merge your pull request. If you don't have permissions, a maintainer will assist.

Thank you for considering contributing to smb-enumaration-lab! We appreciate your efforts and look forward to your contributions.
